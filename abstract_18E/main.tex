\subsubsection*{Project Summary / Abstract}

DOE's 18E challenge is precise: AI tools supporting scientific software development must provide provenance tracking, reproducibility, interpretable performance diagnostics, and uncertainty quantification~\cite{EO14363}. The same Executive Order establishing the Genesis Mission explicitly mandates ``digitization, standardization, metadata, and \textit{provenance tracking}'' as foundational requirements of the American Science and Security Platform. Yet AI systems currently assisting with HPC code development cannot provide these guarantees at the level of individual decisions: they may suggest code optimizations or algorithmic approaches, but cannot trace each recommendation to the specific performance benchmarks, numerical methods literature, or validation results that justify it~\cite{Ji2023HallucinationSurvey}. Without such traceability, AI-assisted HPC code changes cannot be audited, reproduced, or trusted---and in DOE scientific software, numerically incorrect or non-reproducible changes can silently invalidate entire simulation campaigns.

The vision of this project is \textbf{provenance enforcement as a foundational architectural requirement, not a post-hoc annotation, for trustworthy AI-assisted HPC scientific software development}. \textbf{PA-AKG (Provenance-Aware Agentic Knowledge Graph Systems)} builds provenance into the architecture at every layer, targeting the scientific software development lifecycle. At the knowledge representation layer, a persistent KG encodes HPC software knowledge as first-class entities---algorithms, numerical methods, performance benchmarks, hardware constraints, validation results, and development decisions---linked by explicit semantic relations, providing an auditable substrate for AI code recommendations. At the automation layer, \textit{span-level evidence alignment} links every AI suggestion to the specific benchmark result, literature passage, or test outcome that justifies it; \textit{canonicalization auditing} resolves function names, method references, and hardware targets to canonical KG nodes, ensuring consistent cross-session entity resolution. At the agentic orchestration layer, a provenance inheritance constraint requires every agent handoff to carry and extend the accumulated evidence chain, so justifications are never silently discarded as reasoning progresses through multi-step code development workflows.

PA-AKG achieves the 18E trustworthiness requirements through four measurable capabilities. First, \textbf{provenance tracking}: span-level alignment produces a \textit{provenance completeness score}---the fraction of AI recommendations linked to specific supporting artifacts---transforming an aspirational requirement into an auditable, improvable metric. Second, \textbf{reproducibility}: canonical KG nodes ensure algorithm and method references resolve consistently across sessions, code reviewers, and model versions, enabling independent reproduction of AI-assisted software decisions. Third, \textbf{uncertainty quantification}: low provenance completeness scores serve as explicit uncertainty signals, flagging AI recommendations that lack sufficient evidentiary support and enabling risk-aware decision-making before changes enter production code. Fourth, \textbf{interpretable diagnostics}: the E1--E4 evaluation framework operationalizes trustworthiness as actionable measurements---extraction reliability, canonicalization consistency, traceability completeness, and longitudinal stability---surfacing failure modes before they propagate into scientific software rather than returning opaque pass/fail verdicts.

Our prior work establishes both the urgency and the technical path forward~\cite{cusati2026fose, brown2025esem}. The proposed system is designed to integrate with DOE's American Science and Security Platform, contributing provenance-enforcing AI infrastructure applicable across the Genesis Mission's scientific software domains. Primary validation targets SE research workflows; secondary validation focuses on HPC performance analysis at Sandia National Laboratories, directly assessing PA-AKG integration with DOE scientific software development practice. The team spanning Virginia Tech (AI/NLP, software engineering) and Sandia (HPC software infrastructure, provenance engineering) provides the disciplinary coverage required. Key risks---extraction reliability at HPC code scale and KG schema portability across software domains---are mitigated by the staged E1--E4 evaluation design and Sandia's established HPC validation corpora.

\subsubsection*{Research Plan}

\paragraph{Introduction.}
The 18E challenge identifies a structural problem with current AI for scientific software: capability has outpaced accountability. Agentic AI systems can now execute multi-step research workflows, synthesize literature, and generate scientific content at significant scale~\cite{Park2023GenerativeAgents}---but the infrastructure for verifying that their outputs are grounded in correct evidence, reproducible across sessions, and diagnosable when they fail has not kept pace. This is not a problem of model accuracy alone. It is a problem of architecture: current systems are not designed to produce outputs whose correctness can be systematically audited.

The consequences in HPC scientific software are severe. An engineer using AI to port a legacy Fortran simulation to GPU architectures receives code suggestions that may reference documentation but cannot show which benchmark or numerical analysis justifies each transformation. A code reviewer attempting to verify an AI-assisted refactoring cannot trace the performance rationale without re-running the original benchmarks. A system exhibiting systematic failure modes---recommending numerically unstable alternatives, conflating semantically similar but computationally distinct algorithms, or citing validation results that do not support the suggested optimization---has no mechanism to surface these patterns before they propagate into production scientific software~\cite{Ji2023HallucinationSurvey}.

Prior work surveying the agentic AI research landscape identifies three interdependent gaps that jointly produce this accountability deficit~\cite{cusati2026fose}: (a) no existing system tightly integrates persistent structured knowledge representation with autonomous multi-agent research workflows; (b) no validated evaluation framework addresses AI-generated research outputs at the granularity of individual claims and evidence chains; and (c) no generalizable mechanism enforces the traceable linkage between generated claims and specific supporting evidence as an architectural constraint. PA-AKG addresses all three simultaneously, treating them as a cycle that must be broken at the architectural level.

\paragraph{Objectives.}
\begin{enumerate}
    \item Design and implement the PA-AKG three-layer architecture with provenance enforcement as an architectural invariant: knowledge representation, automation and extraction, agentic orchestration.
    \item Implement span-level evidence alignment producing structured claim--evidence pairs with character-level source passage linkage, enabling partial automated verification of claim grounding.
    \item Implement canonicalization auditing with normalization, validation, and deduplication pipelines, ensuring entity resolution is consistent across sessions and reviewers.
    \item Develop the E1--E4 evaluation framework as a quantitative trustworthiness measurement system: extraction reliability (E1), canonicalization consistency (E2), traceability completeness (E3), persistence over time (E4).
    \item Validate that E1--E4 metrics provide actionable diagnostic signals---that low scores on individual axes correspond to identifiable, correctable failure modes in AI-generated scientific content.
\end{enumerate}

\paragraph{Research Questions.}
\begin{itemize}
    \item \textbf{RQ1 (E1):} Does span-level evidence alignment produce claim--evidence links that match human expert annotations, and what fraction of AI-generated claims can be automatically linked to specific supporting passages?
    \item \textbf{RQ2 (E2):} Does canonicalization auditing produce entity cluster assignments that agree with expert judgment (Cohen's $\kappa$), enabling reproducible cross-session entity resolution?
    \item \textbf{RQ3 (E3):} What fraction of knowledge graph nodes maintain complete provenance chains as the research corpus scales, and how does this fraction degrade without architectural enforcement?
    \item \textbf{RQ4:} Do E1--E4 metric scores predict identifiable failure modes in AI-generated scientific content, and do improvements in scores correspond to measurable improvements in output trustworthiness?
\end{itemize}

\paragraph{Detailed Research Plan.}

\textit{Phase 1 (Months 1--3): Architecture Design and Knowledge Representation Layer.}
We will design the KG schema with first-class entity types targeting HPC scientific software workflows: \textsc{Algorithm}, \textsc{NumericalMethod}, \textsc{CodeComponent}, \textsc{HardwareConstraint}, \textsc{Benchmark}, \textsc{ValidationResult}, and \textsc{Evidence}, linked by typed semantic relations (\textsc{implements}, \textsc{requires}, \textsc{supersedes}, \textsc{validates}). Each entity node carries structured attributes including assumptions, constraints, metrics, and evidence spans anchored to source documents via character-level identifiers and quoted text. This architecture supports the 18E requirements directly: provenance tracking (every node carries origin evidence spans), reproducibility (canonical node identifiers are stable across sessions and model versions), interpretable diagnostics (entity relationships expose AI reasoning structure), and uncertainty quantification (provenance completeness score as an explicit confidence signal). We will implement a hybrid symbolic--semantic design pairing a property graph store with a vector index, and establish baseline provenance completeness measurements on an initial corpus to characterize the accountability gap before and after PA-AKG enforcement.

\textit{Phase 2 (Months 4--6): Automation Layer and Provenance Enforcement.}
We will implement the extraction pipeline using structured prompting to produce claim--evidence pairs, followed by span-level alignment assigning character-level source passage references to each extracted claim. Canonicalization auditing normalizes entity mentions through embedding-based clustering, validates cluster assignments against KG-encoded entity constraints, and deduplicates to canonical nodes. The agentic orchestration layer enforces provenance inheritance: each agent receives a provenance context object carrying accumulated evidence chain references, and may only produce outputs that extend---not discard---the inherited chain. Synthesis agents generate hypothesis artifacts with structured provenance metadata; evaluation agents execute reproducible workflows and write results back to the KG as versioned evidence nodes. All agent logs, extraction prompts, KG snapshots, and provenance chains are archived as reproducibility artifacts, extending the reporting protocol of Baltes et al.~\cite{Baltes2025LLM} to the specific case of multi-agent research assistance.

\textit{Phase 3 (Months 7--9): Evaluation Framework and Diagnostic Validation.}
We will evaluate PA-AKG along the E1--E4 axes. \textbf{E1} (extraction reliability): precision and recall of span-level claim--evidence links against 500 human-annotated gold pairs from the SE research corpus. \textbf{E2} (canonicalization consistency): Cohen's $\kappa$ between PA-AKG entity cluster assignments and expert-labeled canonical clusters, establishing inter-rater reliability. \textbf{E3} (traceability completeness): percentage of KG nodes maintaining complete provenance chains as a function of corpus size, compared against a standard RAG baseline without structured provenance. \textbf{E4} (persistence): longitudinal stability score across six corpus update cycles, measuring whether provenance completeness degrades as new evidence enters the system. A diagnostic analysis will characterize failure modes associated with low E-axis scores---identifying whether degradation corresponds to extraction errors, canonicalization failures, or agent handoff violations---to validate that the metrics are actionable.
